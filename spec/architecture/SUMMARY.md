Open-question decisions:
- Strict subset: default-deny whitelist of owned AST node kinds; classes/this/new/generators/symbols/enums banned; throw/try/catch admitted as within-evaluation control flow; async/await is the sole suspension surface; five grammar-enforced capture rules make serializability an admission property (ADR-01).
- Canonical printer/hashing: identity = SHA-256 over owned TLV encoding of the normalized AST (never text); r<n>_ base32 addresses carry the schema version; types+contracts IN the hash, comments/docstrings/local names OUT (De Bruijn locals); hashes immortal via append-only decoders + mutation matrix + nightly world-rehash canary (ADR-02).
- Catalog: five tables — INSERT-only `definition` keyed by hash, `definition_meta`, mutable `name_pointer` (resolution user>team>org>package>product), trigger-written history, `admission` ledger; code+DDL+pointers in one SERIALIZABLE transaction (ADR-03).
- Interpreter: own pure-Go defunctionalized CEK machine from day one (no goja bootstrap, no tsgo emit); suspension anchored (def_hash, node_path, phase); fuel parks as durable condition, never panics; differential conformance vs node; taal AOT-to-Go seam fixed and reserved (ADR-04).
- Continuations: state-capture (not event-sourced replay); C/E/K frames in owned versioned binary TLV ("CFR"); capture verifier at admission (encodable ≡ admitted); conditions+restarts as rows; exactly-once via step_seq CAS + lease; ten-test kill suite gates M0 (ADR-05).
- Kernel: goroutine-per-request (Go netpoller is the reactor); owned PG wire client; immortal-by-hash cache; one SKIP LOCKED task table with lease/heartbeat/reaper (ADR-06).
- Verifier suite v1: six — capability-audit, pii-flow, catalog-parity, contracts, capture, derivation-parity — run against a hermetic in-memory tsgo host (verdict = f(patch, snapshot, epoch)); dual mutation testing + hostile corpus; coverage versioned as rows (ADR-07).
- Epochs: streng's atomic epoch adopted = (kernel binary, std-manifest-root) with boot-refusal on mismatch; migrate-N = dry-run findings-as-rows then one all-or-nothing commit; continuation survival stated as five testable obligations (O5 = the R1-05 per-transaction fleet fence) incl. golden-continuation corpus (ADR-08).
- Git projection: repo = deterministic fold over the admission ledger (byte-identical SHAs are a release gate), stored self-healing mirror; PR merge IS the admission transaction — no side door; overlays never projected (ADR-09).
- std/ admission: std/ IS rows in v1 via a deterministic build-time-canonicalized genesis transaction, but not self-hosting — std bodies are NativeBody nodes dispatching to Go by hash; 14 batteries ship, 5 deferred by Rule of Three (ADR-10).
- Component vocabulary: the ~25 roster is enumerated (25 tier-1 primitives with exactly six PII-masking value leaves; tier-2 derived form/data-table/board/detail/dashboard/operator-plane); no raw-HTML escape hatch exists (ADR-10).
- Reactive layer: admission-time static/dynamic AST split, dynamic slot = diff unit, SSE-down + POST-up, snapshotHash self-healing resync; UI sessions are capped/TTL'd continuation rows; dependency-exact invalidation closed by construction (erf read API is the only read path) (ADR-11).
- Agent plane: 11 MCP tools + 6 resources + 3 prompts, each with abuse-mode→control; agents are ordinary capability principals; per-principal admission-fuel budget; product scope only via one-shot hash-bound human approval token; vault plaintext structurally unreachable to agents; restart buttons resume exactly {conditionId, expectedHash} (ADR-12).

Top 3 risks (RISKS.md): (1) canonical-printer hash identity — a hash-stable normalization bug is invisible to the canary and identity failure is total; (2) continuation serialization — "stable for years" is simulation-proven only, no replay fallback; (3) admission correctness — verifier coverage is enumerative on a shared heap; the harness, not the type system, is the security boundary.
